# DESIGN — Quests and Activities

> Living document. Hand-authored quests, procedurally generated quests,
> ambient activities, and the unified system that makes them
> indistinguishable to the player.
>
> Coupled to *Content Pipeline* (authoring), *World Simulation* (firing
> triggers), *NPCs and Companions* (quest givers/targets), *Factions
> and Reputation* (faction quests), and *Crime and Justice*.

---

## 1. Stance

The game must support:
- **All canonical Pathfinder Adventure Paths, Modules, and Society
  Scenarios** (per *Rules and Edition* §4.1).
- **Thousands of original / procedural / sandbox / ambient
  activities.**

Both flow through one **unified quest system**. The player should
not be able to tell whether a "deliver this letter to the smith"
quest came from an authored AP or was procedurally composed; both
have the same affordances (journal entry, NPC dialogue hooks,
faction interactions, possible failure consequences).

## 2. Quest Anatomy (proposed)

```
Quest
├── identity (id, title, source)
├── triggers (how does it start? — AP date, NPC interaction,
│             location entry, world event, declared interest,
│             rumor heard, item found, …)
├── beats (ordered or DAG of objectives, each with success/fail
│          tracks)
├── actors (NPCs / factions / locations involved)
├── consequences (what changes in the world as it advances/ends)
├── rewards (xp, gold, reputation, items, body state, romance,
│             knowledge, follow-up quest unlocks)
└── failure modes (timeout, betrayal, key-NPC death, missed window)
```

## 3. Quest Tiers

| Tier | Origin | Examples |
|---|---|---|
| **Major** | Hand-authored | AP arcs, central plot, romance ladders |
| **Side** | Hand-authored | Module quests, faction quests, side stories |
| **Procedural** | Generated by templates + world state | "Bandit raid on village X", "missing-person at lodge Y" |
| **Ambient** | Lightweight, autonomous | "Drink with Erik at the Rusty Nail tonight", chores |
| **Self-set** | Player-declared goals | "Become a master smith" |

## 4. Procedural Composition

Procedural quests use **templates + world state**:
- A template is a quest skeleton with parameter slots
  (location, NPC role, faction, item, body state, etc.).
- The composer fills slots with currently-relevant world entities,
  respecting personality, faction relationships, geography.
- The result is committed to the live quest log as if hand-authored.

## 5. Adventure Path Integration

- Each AP is a curated quest tree imported via the *Content
  Pipeline*.
- AP firing rules per WS-Q18.
- Concurrent APs handled by simulation (WS-Q20).

## 6. Cross-References

## 7. Open Questions (Quests-Local)

- ~~**Q-Q1**~~ **Resolved (mirrors CP-Q1).** Hybrid:
  Markdown-with-YAML-frontmatter for prose-heavy quest content
  (descriptions, dialogue, scene beats); pure YAML for
  stat/structure-heavy data (objectives, triggers, rewards,
  encounters).
- **Q-Q2** Quest journal UX — diegetic (in-game journal book) vs
  modern menu vs hybrid.
- **Q-Q3** Quest discovery — does the player see "active quests" or
  must they remember (Morrowind-style)?
- **Q-Q4** Markers — quest markers on map / minimap / compass /
  none?
- **Q-Q5** "Quest" as a noun — does the in-game world use the word
  "quest" or only "task / errand / contract / deed / ..."?
- **Q-Q6** Procedural template count target — 100s, 1000s?
- **Q-Q7** Procedural quality bar — when does a generated quest get
  scrapped before it's offered (e.g., absurd combos)?
- **Q-Q8** Self-set goals — does the game help the player track
  them, or are they purely informal?
- **Q-Q9** Multi-AP concurrent play — does the player see them as
  separate "campaigns" or all one journal?
- **Q-Q10** Failure consequences — how irreversible? How loud is
  the feedback?
- **Q-Q11** Time-windowed quests — "deliver before nightfall" —
  enforced by simulation, or fudged?
- **Q-Q12** Quest chain branching — DAG? linear? full graph?
- **Q-Q13** Romance quests — handled the same way or special-cased?
- **Q-Q14** NSFW quests — same affordances; any special handling?
- **Q-Q15** Quest-NPC death mid-quest — what happens?
- **Q-Q16** Player abandonment — can the player formally abandon a
  quest, or does it just rot in the journal?
- **Q-Q17** Reward scaling — fixed PF2e treasure-by-level, or
  context-driven?
- **Q-Q18** AP triggers vs player choice — does the AP "find" the
  player, or does the player "find" the AP?
- **Q-Q19** Society Scenario UX — packaged as Pathfinder Society
  membership in-fiction, or as a pickable "Society quest" tag?
- **Q-Q20** Quests authored by AI dev-time — at what step?
- **Q-Q21** Activity vs quest — what differentiates a "side activity"
  (gambling, fishing) from a "quest"? Both supported, but UX-distinct?
- **Q-Q22** Daily / repeatable quests — supported, capped, none?
- **Q-Q23** Quest difficulty / level scaling — fixed PF2e CR, soft
  scaling, hard scaling?
- **Q-Q24** Cross-quest interactions — Quest A's outcome affects
  Quest B (side-effect graph)?
- **Q-Q25** Hidden quests — discoverable only via investigation?
